Buenas prácticas de edición de código en JavaScript

1- No utilizar JavaScript de forma intrusiva.

Una buena práctica es utilizar JavaScript de forma no intrusiva debido a que este queda separado del documento HTML haciendo que este pueda ser más fácil de editarlo por que como queda separado del documento anterior mencionado podamos tener mucho más organizado el código facilitando más las cosas.

Entonces, el Javascript no intrusivo, se fundamenta en lo hablado anteriormente, es decir, en separar la funcionalidad cliente de una página del contenido, ya que a fin de cuentas un documento HTML, no es otra cosa que un documento que nos permite difundir información en un medio (si no, preguntádselo a google) de manera que este, tiene que gozar de una estructura de marcación que nos permita conseguir ese fin..

Java Script no intrusivo

document.getElementById("btn").onclick= function holaMundo() {
alert ("Hola Mundo!");
}

2- Comentar el código de JavaScript.

Es muy recomendable ir comentando el código de lo que estamos haciendo para especificar para que funciona tal sintaxis y es bastante útil cuando se trabaja en equipo por que con eso se puede ir especificando a los miembros del equipo cómo funciona y que cambios necesita

Puede parecer innecesario al principio, pero, es de gran importancia comentar tu código lo mejor posible. Es una fase muy importante para recordar nuestro pensamiento a la hora de resolver el problema ademas de que no ayuda a darle mantenimiento en caso de ser un proyecto muy grande, ya que siempre se cuenta con un equipo de trabajo y es importante saber que hace cada parte del código.

Ejemplo de comentarios

var multipli=(x,y)=>x*y;
//Esta función flecha retorna un resultado de la multiplicacion 2 variables
alert(multipli(3,3));

comentario multilinea*/.

3- Colocar los scripts en la parte inferior de la pagina.

La idea de esto es que la pagina se cargue lo más rápido posible para el usuario esto se debe mas que todo por que el navegador debe de cargar todo el contenido antes.

4- Utilizar comas en lugar de inicializar var en una lista larga

Si se posee una lista larga de variables hay que omitir la palabra clave var y utilizar comas en su lugar haciendo que este funcione de igual manera haciendo esto una práctica útil de utilizar.

Esta parece una práctica no tan útil pero en realidad es muy importante ya que siempre a la hora de declarar variables, a diferencia de lenguajes como C, C# y C++, no es necesario que declaremos el tipo de dato, por lo cual podemos declarar variables de diferente tipo usando una sola vez la palabra revervada var. Por ejemplo.

Ejemplo de declaración de variables.

var num, nombre, fechaNac, password;
//Con esta declaracion básica de variables ahorramos código, de la otra forma no recomendable quedaria:
var num;
var nombre;
var fechaNac;
var password;

5-Declara variables fuera de la sentencia For

Cuando se ejecuta una larga sentencia "for", hay que evitar hacer trabajar al motor más de lo necesario. Por ejemplo:

Forma incorrecta
for(var i = 0; i < varible.length; i++) {
var container =document.getElementById ('container');
container.innerHtml += 'my number: ' + i;
console.log(i);
}
Forma recomendada
var container =document.getElementById ('container');
for(var i = 0; i < varible.length; i++) {
container.innerHtml += 'my number: ' + i;
console.log(i);
}
6- Usar siempre el punto y coma.

Técnicamente, la mayoría de los navegadores permitirán omitir los puntos y comas.
Dicho esto, se trata de una práctica muy mala que puede dar lugar a problemas mucho mayores y más difíciles de encontrar.

7- No abusar del uso de las bibliotecas de JavaScript.

Las bibliotecas de JavaScript, como jQuery y Mootools, pueden ahorrarte una enorme cantidad de tiempo a la hora de codificar, especialmente con las operaciones AJAX. Dicho esto, ten siempre presente que una biblioteca nunca puede ser tan rápida como el JavaScript en puro siempre y cuando se escriba correctamente.

El método "each" de jQuery es muy bueno para hacer bucles, pero usar una sentencia "for" nativa siempre será un poco más rápido.

8- El uso de comillas.

Hay algunas recomendaciones acerca de esta notación. Aunque Javascript permite usar comillas simples o comillas dobles indistintamente, generalmente es recomendable, en ciertos casos, escribir con comillas simples y en otros con comillas dobles.

En ciertos casos se desea tener comillas dentro de un texto, y tendrán otras como delimitadores externos del texto.

9- Asingar nombre con sentido a las variables.

La intención de asignar un nombre con sentido a carpetas, archivos, variables, funciones, etc. Es indicar cuál es su cometido, de tal manera que quede claro el propósito por el cuál fue creado. Y es que esta tarea se realiza constantemente, y toma su tiempo porque muchas de las veces cuesta mucho saber que nombre se le puede asignar para indicar de forma directa su intención y cometido. Pero también tiene sus beneficios asignar nombres coherentes, puesto que más adelante será menor el esfuerzo cuando se vuelva a leer el código pues facilita su compresión.

10- Usar el operador ternario.

El operador ternario sirve para escribir el código de una sentencia if de una forma mucho más sencilla. Lo anterior resulta en un código más limpio y fácil de leer.
Este operador es el único en JavaScript que toma tres operandos:
- La condición seguida del signo de interrogación ?.
- Una expresión a ejecutar si la condición es verdadera seguida de dos puntos :
- La expresión a ejecutar si es falsa.
